Class 12 Computer Science - DBMS Summary

Detailed interactive guide to understand Database Management Systems

Data vs Information

Data are raw, unorganized facts and figures that do not carry any specific meaning alone. For example, names, numbers, and places like "Ram, 17, Pokhara" are considered data.

Information is processed and organized data that provides meaningful context. For example, "Ram is 17 years old and lives in Pokhara."

Remember: All information is data, but not all data is information.

What is a Database?

A Database is a systematic collection of interrelated data that can be easily accessed, managed, and updated. It's organized in tabular form and is useful for data retrieval and decision-making processes.

Drawbacks of File-Based System

  • Data Redundancy: Duplicate data across multiple files
  • Lack of Security: Limited control over access
  • Complex Data Retrieval: Hard to search and generate reports
  • Slow Performance: Inefficient for large data

Database Management System (DBMS)

A DBMS is software that interacts with end users, applications, and the database itself to capture and analyze data.

Advantages of DBMS

  • Sharing of data among multiple users
  • Reduced data redundancy
  • Backup and recovery features
  • Data security using access control
  • Multiple user interface support
  • Data integrity and consistency

Disadvantages of DBMS

  • High cost of installation and maintenance
  • Requires skilled manpower
  • Frequent backups needed

Database Models

Hierarchical Model

Tree-like structure with parent-child relationships. Example: An organization chart.

Advantages:

  • Simple structure
  • Efficient for 1:1 or 1:many relationships

Disadvantages:

  • No support for many-to-many relationships
  • Redundant data entries

Network Model

Data is organized as records connected by links. Allows many-to-many relationships.

Advantages:

  • Flexible data access
  • Reduced redundancy

Disadvantages:

  • Complex structure
  • Requires detailed programming

Relational Model

Data is stored in tables (relations). Widely used today. Example: MySQL, MS Access.

Advantages:

  • Efficient querying
  • Supports complex relationships

Disadvantages:

  • Complex for very large systems
  • Requires powerful hardware

Structured Query Language (SQL)

SQL is a standard language for managing relational databases. It includes:

  • DDL: Data Definition Language (e.g., CREATE, ALTER)
  • DML: Data Manipulation Language (e.g., SELECT, INSERT, UPDATE, DELETE)
  • DCL: Data Control Language (e.g., GRANT, REVOKE)

Example: CREATE TABLE Student (ID INT, Name TEXT);

Entity Relationship Diagram (ERD)

  • Entities: Objects like Student, Teacher
  • Attributes: Properties like Name, ID
  • Relationships: Associations like "teaches", "enrolls"

ER diagrams are used for database design and understanding relationships between entities.

Database Administrator (DBA)

Qualities

  • Skilled in DBMS tools
  • Expert in network and OS
  • Good database design and backup knowledge

Responsibilities

  • Database installation and configuration
  • Managing user roles and privileges
  • Ensuring security and backup

Normalization

Process of organizing data to eliminate redundancy and dependency. Introduced by E.F. Codd in 1970.

1NF (First Normal Form)

  • Eliminates repeating groups
  • Ensures atomic values

2NF (Second Normal Form)

  • Based on 1NF
  • Eliminates partial dependency

3NF (Third Normal Form)

  • Based on 2NF
  • Eliminates transitive dependency

Advantages of Normalization

  • Reduces redundancy
  • Improves data consistency
  • Efficient indexing and sorting
  • Better performance
